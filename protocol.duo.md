一、通信说明
1、BLE蓝牙透传。
2、采用主从通讯结构，手机端为主机，仪器端为从机，主机500mS查询1次，从机应答。
二、指令详细说明
1、主机端指令，查询从机信息 
帧头
1Byte	长度
1Byte	命令
1Byte	和校验
1Byte
0xA5	0x02	0xC0	add
帧头固定：0xA5
长度：0x02，表示后有2个字节数据
命令：固定0xC0；
和校验：命令+数据=add，无数据，发送 0xC0。

命令示例：0xA5,0x02,0xC0 ,0xC0

1、从机端应答，回复从机信息
帧头
1Byte	长度
1Byte	命令
1Byte	数据
11Byte	和校验
1Byte
0xA5	0x0D	0xC0	XXX XX XX XXXX	add
帧头固定：0xA5
数据长度：0x0D，表示后有13个字节数据
命令：固定0xC0
数据1\2\3：A板版本X.X.X，3个字节
数据4\5：CH1板版本X.X，2个字节
数据6\7：CH2板版本X.X，2个字节
数据8\9\10\11：仪器累计工作时间XXXX，32位数据，低位在前，高位在后。

和校验：命令+数据1+数据2+数据3+~数据11=add，去掉高字节只保留低字节，如：0x015E只发送 0x5E。

命令示例：A板版本1.0.0，CH1板版本1.0，CH2板版本1.0，仪器累计工作时间5000分钟
0xA5,0x0D, 0xC0,0x01,0x00,0x00,0x01,0x00,0x01,0x00,0x88,0x13, 0x00, 0x00,0x5E




2、主机端指令，下发参数
帧头
1Byte	长度
1Byte	命令
1Byte	数据
8Byte	和校验
1Byte
0xA5	0x0A	0xB0	X XX XXX XX	add
帧头固定：0xA5
长度：0x0A
命令：固定0xB0；
数据1：诊所选择，0x00为非诊所界面、0x01为选择诊所界面。
数据2：通道选择，0x00为CH1，0x01为CH2。
数据3：模式选择，0x00为无阻尼，0x01为有阻尼。
数据4：频率设置，0x01~0xC8，1~200为0.1Hz~20.0Hz，扩大10倍发送。
数据5：强度设置，0x00~0x64，0~100为0~100%。
数据6：时间分，0~60
数据7：时间秒，0~59
数据8：启动/暂停状态，0x00 暂停，0x01启动。
和校验： add

帧头
1Byte	长度
1Byte	命令

1Byte

数据4：频率设置，0x01~0xC8，1~200为0.1Hz~20.0Hz，扩大10倍发送。
数据5：强度设置，0x00~0x64，0~100为0~100%。
数据6：时间分，0~60
数据7：时间秒，0~59
数据8：启动/暂停状态，0x00 暂停，0x01启动。
数据9\10\11\12：诊所界面下，累计总能量显示XXXX，单精度浮点数，低位在前，高位在后。
和校验： add
